# Roadmap

This roadmap describes the intended evolution of ConnectHub from documentation-first foundation to a portfolio-ready cloud native system.

## Phase 0 — Foundation

Status: In progress

- Establish the documentation baseline.
- Define project scope and architecture direction.
- Confirm service naming and repository conventions.

## Phase 1 — Core platform skeleton

Status: Planned

- Create the initial application structure.
- Introduce shared modules where needed.
- Add container build support.
- Document local development workflows.

## Phase 2 — First functional services

Status: Planned

- Implement the first domain service.
- Add authentication integration with Keycloak.
- Establish persistence, health checks, and metrics.
- Publish the first service contracts.

## Phase 3 — Multi-service platform

Status: Planned

- Expand to additional ConnectHub services.
- Document service interactions and data ownership.
- Introduce observability dashboards.
- Add Kubernetes deployment assets and overlays.

## Phase 4 — Production-inspired hardening

Status: Planned

- Refine resource management and failure handling.
- Improve security, release, and rollout documentation.
- Validate OpenShift compatibility.
- Add operational runbooks and troubleshooting notes.

## Phase 5 — Portfolio polish

Status: Planned

- Review the repository for clarity and public presentation.
- Tighten documentation and architecture narratives.
- Add diagrams and decision records where useful.
- Ensure the project tells a coherent story for reviewers.

## Open items

- Final service backlog
- API contract definitions
- CI/CD pipeline design
- Diagram set in `docs/architecture/diagrams/`
- Implementation milestones for each service
